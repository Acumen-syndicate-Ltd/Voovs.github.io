How do we do all this site stuff?

If you'd like more specific points, feel free to email us through the "Contact" button on the header. If that doesn't work, try sending it to acumenofknowledge@gmail.com

We start off with the basic code development through CodePen. It's a pretty great site for web development, as it has colours and a live preview. Note that's it's also free. After the initial development, we place it in our index.html file. This file is linked to a Github repository, which is pretty difficult to set up for the first time. We'd recommend watching a few recent Youtube tutorials for this part. The process is surprisingly specific, so don't just try to get it on your own. After having been synchronized with Github, the site goes through Netlify. This is a free service which will allow you to get free HTTPS, instead of normal HTTP. That "S", at the end, highly improves your site's security on the client side. Now, Netlify will also give you a DNS, though it's randomized, so the results are pretty ridiculous. The site that you're on right now got the DNS http://quirky-raman-8c1296.netlify.com/, which isn't ideal for most folks. A custom DNS, like our Acumensyndicate.ca, needs to be obtained through a domain registry. Usually, these do come with a yearly cost. Freenom distributes free DNSs, though we can't certify how well they perform, having never used it. Freenom also has the off-putting trait of not having HTTPS on their own site! We used Godaddy, a registry that does charge for domains. This particular site had a pretty meaty discount on .CA domains, so that was our motivator. Regardless of what registry you choose, Netlify will practically give you a tutorial about how to connect it; another reason for using Netlify in the process. Now your site should be up and running. All that's left to do is learn some HTML/CSS/JS. Scroll down to see our recommendations

Where do you learn web development languages?

Honestly there are a lot of options for starting off with web development. We'll list the better ones below then dive into deeper explanations. Note all of these are free

  • -MDN
  • -W3Schools
  • -Codecademy
  • -Freecodecamp

MDN- Acronym for Mozilla Developer Network, this site provides a very presentation-like learning experience. If you have little patience, this isn't the option for you. Those who like to take notes and read through long book may be more inclined towards this one. Effectively, these are just really long pages of explaining how aspects of the code work. There are little activities throughout these pages, for more practical experience. There are also assessments for each unit; very similar to modern education structure. The key feature about MDN is how they don't really tell you exactly how to code. Instead they aim for comprehension of the whole language. All the element tags and selectors aren't really taught in the course, though through lots of exposure, they just become second nature. I, Voovs, personally use this site for learning stuff, yet most others really dislike the immense patience required for this one

W3Schools- Quickly learning small bits of information is the strategy of this site. If you like slow and steady comprehension of the code, this site is most definitely not for you. Some would hardly consider this a learning platform, as it's just an explanation of every tag, selector, value, and technique found in web development. However, there are people the appreciate this sort of learning. Commonly these are the ones who want to get straight to web development without learning the theory acpet first. There's absolutely no problem with being like that, so try this site out. The site is also very good at solving problems when you don't know a selector like "::selection" or something like that. Unknown prefers this site for learning code, as will many of those who just want to start web development

Codecademy- A code-to-learn-code sort of system. If you've ever used Swift playgrounds, it's very similar in layout. Generally most people learn from this site, though it does have a restrictive nature to it. Think of a video game. There are different levels, in this case lessons, with only so much you can do in each one. If you'd like to go through this guided practice style of learning, Codecademy might be for you. A side note about this one is that you can learn more than just web development here

Freecodecamp- Think of a long list with sort of units. That's kind of what Freecodecamp is. The actual learning layout is comparable to that of Codecademy. Most people who don't use Codecademy use Freecodecamp. They are truly very similar in nature, though rumour has it that Freecodecamp is a bit more in depth. If you're into the layout and guided practice learning style of Codecademy, Freecodecamp is an alternative I'd highly recommend you check out. Maybe you'll like the phrasing of concepts a bit more.
